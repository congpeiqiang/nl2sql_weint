# 主智能体路由技能

## 概述

主智能体是系统的入口，负责理解用户意图并将任务委派给对应的子智能体。本技能定义意图识别规则和委派策略。

## 动态发现规则

> **技能和工具会更新，禁止依赖硬编码列表。** 被问到子智能体详情时，
> 必须从 YAML 配置文件实时读取：
>
> ```bash
> cat subagents/configs/nl2sql.yaml     # 读取配置
> ls subagents/configs/                 # 列出所有子智能体
> ```
> 回答时提取 `description`、`skills`、`tools` 字段。

## 核心职责

1. **意图识别** — 分析用户输入，判断任务类型
2. **子智能体选择** — 根据意图选择合适的子智能体
3. **委派执行** — 将任务委派给子智能体并收集结果
4. **结果呈现** — 以友好格式展示子智能体返回的结果

## 可用子智能体清单

### nl2sql — NL2SQL 数据查询专家

| 维度 | 详情 |
|------|------|
| **职责** | 将用户自然语言数据问题转换为 SQL 并执行，支持图表可视化 |
| **技能** | `sql-of-thought`（编排器）、`nl2sql-schema-linking`（表发现）、`nl2sql-subproblem`（分解）、`nl2sql-query-plan`（计划）、`nl2sql-sql-generation`（生成）、`nl2sql-correction`（纠错） |
| **工具** | `run_sql`（执行SQL）、`suggestCharts`（推荐图表）、`getSchema`（图表Schema）、`renderChart`（渲染图表） |
| **Shell** | `wren context show`（查看表模型）、`wren dry-plan`（验证SQL）、`wren query`（查询）、`python skills/sql-of-thought/scripts/gen_models_mysql.py`（生成模型） |

## 意图分类

### 数据查询 → 委派 `nl2sql`

**触发关键词：** 查询、统计、有多少、列出、排名、对比、计算、汇总、分析、数据、SQL、图表、可视化、报表

**示例：**
- "查询销售额最高的10个产品" → nl2sql
- "统计各部门员工数量" → nl2sql
- "分析客户购买行为" → nl2sql
- "画一张销售趋势图" → nl2sql

### 数据库管理 → 委派 `nl2sql`

**触发关键词：** 建表、导入数据、修改表结构、Schema、数据库配置、连接

**示例：**
- "切换到 Chinook 数据库" → nl2sql（传递 db_name）
- "查看数据库中有哪些表" → nl2sql

### 一般对话 → 直接回复

**触发关键词：** 你好、帮助、功能、你是谁、能做什么、谢谢

**示例：**
- "你好" → 直接回复
- "你能做什么" → 介绍功能

## 委派策略

### 单步委派
大多数数据查询只需委派给 nl2sql 一次：
```
用户: "查询销售额"
  → 主智能体识别: 数据查询
  → 委派 nl2sql
  → 返回结果
```

### 多步委派（暂未实现）
未来可支持：
```
用户: "分析销售数据并生成报告"
  → 主智能体识别: 数据查询 + 报告生成
  → 先委派 nl2sql 查询
  → 再委派 report 生成
```

## 状态传递

委派时传递以下信息给子智能体：
- `config.configurable.db_name` — 当前选中的数据库
- `config.configurable.user_id` — 用户身份
- 原始用户问题

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 子智能体执行失败 | 向用户解释错误，建议解决方案 |
| 意图不明确 | 询问用户澄清需求 |
| 无匹配子智能体 | 告知用户当前不支持该功能 |

## 与子智能体的关系

- 主智能体 **不执行** 具体的数据查询或分析
- 主智能体 **不持有** 数据库连接或 SQL 工具
- 主智能体 **仅负责** 路由和协调
- 子智能体 **独立拥有** 完成其任务所需的全部工具和技能
